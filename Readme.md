## 📌 Описание проекта
  
Цель работы — получение практических навыков анализа данных (EDA), построения пайплайнов с использованием DVC и реализации различных моделей машинного обучения для задачи регрессии (предсказание уровня смертности от рака).
В рамках проекта выполнены:
- Первичный анализ данных и отбор признаков
- Построение DVC-пайплайна
- Обучение и оценка пяти моделей:
  - Линейная регрессия
  - Дерево решений
  - CatBoost
  - XGBoost
  - Нейронная сеть (MLP)
- Сравнение метрик и анализ переобучения
## 📊 Данные
**Источник:** [OLS Regression Challenge](https://data.world/nrippner/ols-regression-challenge) (сводные данные из American Community Survey, clinicaltrials.gov, cancer.gov)
**Целевая переменная:**  
`TARGET_deathRate` — средняя смертность от рака на 100 000 населения.
**Основные признаки (после EDA):**
- `incidenceRate`, `avgAnnCount`, `avgDeathsPerYear`
- `medianIncome`, `povertyPercent`, `studyPerCap`
- `MedianAge`, `PctEmployed16_Over`, `PctPrivateCoverage` и др.
Всего после предобработки и отбора признаков (корреляция > 0) осталось несколько десятков признаков. Пропуски заполнены медианными значениями.

## 🔍 Exploratory Data Analysis (EDA)
- Построены гистограммы распределения целевой переменной
- Оценена корреляция признаков с целевой переменной (отобраны признаки с корреляцией > 0)
- Построена тепловая карта корреляций между признаками
Результаты EDA позволили сократить датасет и выделить наиболее информативные признаки.

## ⚙️ DVC пайплайн
Для автоматизации и воспроизводимости эксперимента построен пайплайн с использованием **Data Version Control (DVC)**.
Структура:
- `prepare` — загрузка и подготовка данных
- `model_train` — обучение моделей (линейная регрессия, дерево, CatBoost, XGBoost, MLP)
- `model_validate` — валидация на отложенной выборке
- `model_test` — финальное тестирование и сохранение метрик
Пайплайн позволяет перезапускать только изменённые этапы и кэшировать результаты.

## 🧠 Модели и результаты
Все модели обучены на одних и тех же тренировочной/валидационной/тестовой выборках.  
Оценка качества проводилась по метрикам: **MSE, MAE, RMSE, R²**.
### Сравнительная таблица метрик (на тестовой выборке)
| Модель              | MSE     | MAE   | RMSE   | R²    |
|---------------------|---------|-------|--------|-------|
| Линейная регрессия  | 399.961 | 14.58 | 19.999 | 0.467 |
| Дерево решений      | 489.669 | 16.32 | 22.128 | 0.347 |
| **CatBoost**        | 275.884 | 12.42 | 16.609 | 0.632 |
| **XGBoost**         | 244.255 | 11.82 | 15.628 | 0.674 |
| MLP (нейронная сеть)| 506.421 | 16.65 | 22.503 | 0.324 |
### 🏆 Лучшая модель: **XGBoost**
- Наивысший R² = 0.674
- Наименьшая абсолютная ошибка MAE = 11.82
- Однако наблюдается сильное переобучение (R² на тренировке = 0.975)
### Другие выводы по моделям:
- **CatBoost** также показал хороший результат (R² = 0.632), но переобучен ещё сильнее (R² train = 0.962)
- **Линейная регрессия** дала стабильные результаты без переобучения, но качество ниже
- **Дерево решений** и **MLP** показали худшее обобщение (R² < 0.35)
- Для улучшения MLP необходима более тщательная настройка гиперпараметров и увеличение объёма данных
